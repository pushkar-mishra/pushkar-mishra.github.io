§ 01

Foundations of AI and LLMs

The Hierarchy: AI → ML → Deep Learning → Generative AI

These terms are often used interchangeably but represent a nested hierarchy of concepts. Artificial Intelligence (AI) is the broadest category — any technique that enables machines to exhibit intelligent behavior, including rule-based systems, search algorithms, and expert systems. Machine Learning (ML) is a subset of AI where systems learn patterns from data rather than following explicit rules. Deep Learning (DL) is a subset of ML that uses neural networks with many layers to learn hierarchical representations. Generative AI is a class of deep learning models capable of generating new content — text, images, code, audio — by learning the underlying distribution of training data.

Neural Networks: The Foundation

A neural network is a computation graph composed of layers of parameterized linear transformations followed by nonlinear activation functions. Each "neuron" computes a weighted sum of its inputs, adds a bias term, and passes the result through an activation function like ReLU or GELU. Stacking these layers allows networks to learn increasingly abstract features: pixels → edges → shapes → objects, or characters → words → phrases → semantics.

output = activation(W · input + b) W ∈ ℝ^(out×in), b ∈ ℝ^out

Training vs. Inference

Training is the optimization phase: given labeled or unlabeled data and a loss function, the model's parameters (weights) are iteratively adjusted via gradient descent to minimize prediction error. This is compute-intensive and typically done offline. Inference is the deployment phase: given fixed weights, the model makes predictions on new inputs. Inference can happen in real time and requires lower (but still significant) compute for large models.

Learning Paradigms

Supervised learning trains on input-output pairs (e.g., image classification with labels). Unsupervised learning finds structure in unlabeled data (e.g., clustering, dimensionality reduction). Self-supervised learning — the paradigm powering LLMs — generates its own supervision signal from raw data. For language models, this means predicting the next token given all previous tokens, using the data itself as the label. This allows training on internet-scale text without manual annotation.

§ 02

Transformer Architecture

Motivation: The Failure of Sequence Models

Before Transformers, sequence modeling relied on Recurrent Neural Networks (RNNs) and their gated variants — LSTMs and GRUs. These models process tokens sequentially, maintaining a hidden state that is updated at each time step. The fundamental problems: (1) vanishing/exploding gradients make it difficult to propagate learning signals over long sequences; (2) sequential computation prevents parallelization, making training slow; (3) long-range dependencies degrade because information must flow through every intermediate hidden state.

The 2017 paper "Attention Is All You Need" (Vaswani et al.) proposed discarding recurrence entirely and building sequence models purely from attention mechanisms — allowing every token to directly attend to every other token in a single layer, enabling full parallelization and constant-path-length dependency modeling.

Encoder-Decoder Architecture

The original Transformer was designed for sequence-to-sequence tasks (e.g., translation). The Encoder reads the full input sequence and produces a rich contextual representation of each token. The Decoder takes the encoder's output and autoregressively generates the output sequence one token at a time, attending to both previously generated tokens (via self-attention) and the encoder output (via cross-attention). Modern LLMs largely drop the encoder and use decoder-only architectures.

Self-Attention Mechanism

Self-attention answers the question: "For each token in the sequence, how much should it attend to every other token?" This allows the model to resolve coreferences, capture syntactic structure, and propagate semantic information regardless of distance.

For each token, three vectors are computed from the token's embedding via learned linear projections: a Query (Q), a Key (K), and a Value (V). Think of it like a soft database lookup: the Query is what you're looking for, the Keys are the index entries, and Values are the actual content retrieved.

Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V

The dot product QKᵀ produces attention scores — a matrix of how compatible each query is with each key. Dividing by √d_k (the square root of the key dimension) prevents the dot products from growing too large in high-dimensional space, which would push the softmax into saturation. The softmax converts scores to a probability distribution, and the weighted sum over Values produces the output: a context-aware representation of each token.

Multi-Head Attention

Running a single attention operation captures one type of relationship. Multi-head attention runs H parallel attention operations (heads), each with its own Q/K/V projections, then concatenates and linearly projects the results. Different heads learn to attend to different relationship types: one head might track subject-verb agreement, another might resolve pronoun references, another might capture positional proximity.

MultiHead(Q,K,V) = Concat(head₁,...,headₕ)Wᴼ headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)

Positional Encoding

Self-attention is inherently permutation-invariant — it treats the input as a set, not a sequence. To inject position information, a positional encoding is added to each token's embedding before the first Transformer layer. The original paper used sinusoidal functions of different frequencies. Modern models use learned positional embeddings or more sophisticated schemes like Rotary Position Embedding (RoPE), which encodes relative positions directly into the attention computation.

Feed-Forward Networks

Each Transformer block contains a two-layer feed-forward network applied identically to each token position. This is where much of the model's "knowledge" is stored — research suggests FFN layers act as key-value memories. The hidden dimension is typically 4× the model dimension, making FFNs the dominant contributor to parameter count in most architectures.

Residual Connections and Layer Normalization

Both the attention sublayer and the FFN sublayer are wrapped in a residual (skip) connection: the layer's output is added to its input. This preserves gradient flow during backpropagation and allows layers to learn residual corrections rather than full transformations. Layer normalization (applied before or after each sublayer, depending on the variant) stabilizes training by normalizing activations across the feature dimension.

§ 03

Variants of Transformer Models

Encoder-only (e.g., BERT) Bidirectional attention — each token attends to all others. Optimal for classification, NER, embeddings. Uses Masked Language Modeling: predict randomly masked tokens using full context.
Decoder-only (e.g., GPT, LLaMA) Causal (unidirectional) attention — each token attends only to preceding tokens. The dominant architecture for generative LLMs. Uses Next Token Prediction as the training objective.
Encoder-Decoder (e.g., T5, BART) Full encoder reads input; decoder generates output with cross-attention to encoder states. Best for translation, summarization, seq2seq tasks.
Autoregressive vs Masked Autoregressive: generate left-to-right, conditioning on all previous tokens. Masked: bidirectional prediction during training. Autoregressive models are standard for text generation.
§ 04

Tokenization and Vector Representation

Tokens vs. Words

LLMs do not operate on raw text or individual characters. They operate on tokens — subword units that balance vocabulary size against coverage. A word like "unbelievable" might be tokenized as ["un", "believ", "able"]; "GPT-4" might be ["G", "PT", "-", "4"]. A typical vocabulary contains 32,000–128,000 tokens. The choice of tokenization scheme significantly impacts model behavior, especially for code, non-Latin scripts, and numbers.

Subword Tokenization Algorithms

Byte Pair Encoding (BPE) starts with individual characters and iteratively merges the most frequent adjacent pairs to build a vocabulary. Used by GPT models. WordPiece (used in BERT) is similar but maximizes the likelihood of the training data under a language model. SentencePiece (used in LLaMA, T5) works directly on raw text without pre-tokenization, making it language-agnostic and robust to whitespace variations.

Embeddings

Each token ID is mapped to a dense vector via an embedding lookup table — a matrix of shape (vocab_size × d_model). This converts discrete token IDs into continuous vectors that the model can process mathematically. For GPT-3, d_model = 12,288; for smaller models like GPT-2 small, d_model = 768.

Analogy: Think of the embedding matrix as a giant dictionary where each word has a GPS coordinate in a high-dimensional space. Words with similar meanings cluster together; relationships like "king − man + woman ≈ queen" emerge as vector arithmetic.

Word Embeddings vs. Contextual Embeddings

Classical word embeddings like Word2Vec assign a single fixed vector to each word regardless of context. "Bank" gets one vector whether you mean a riverbank or a financial institution. In Transformers, embeddings are contextual — the representation of a token is dynamically shaped by all other tokens in the sequence via the attention mechanism. The same token at different positions in different sentences produces different internal representations.

High-Dimensional Embedding Space

Models operate in spaces of hundreds to thousands of dimensions. This high dimensionality is not coincidental — it allows the model to encode many independent features simultaneously without interference (the "Johnson-Lindenstrauss" effect). Semantic clusters form naturally: languages cluster, code syntax clusters, factual domains cluster.

§ 05

Data Flow Inside an LLM

Step-by-Step Forward Pass

1. Input Prompt. The user's raw text string enters the system.

2. Tokenization. The text is split into tokens by the tokenizer, producing a sequence of integer IDs. "Hello world" → [9906, 995].

3. Embedding Layer. Each token ID is looked up in the embedding matrix, yielding a matrix of shape (seq_len × d_model).

4. Positional Encoding. A positional encoding vector is added element-wise to each token embedding, injecting sequence order information.

5. Transformer Layers. The sequence passes through N stacked Transformer blocks (e.g., 96 for GPT-3). In each block: multi-head self-attention computes contextual representations, residual + LayerNorm stabilizes, then the FFN applies position-wise transformations.

6. Attention Calculations. Within each attention head, Q/K/V projections are computed for all positions simultaneously. The attention matrix (seq_len × seq_len) is computed via scaled dot-product. In decoder-only models, a causal mask zeros out attention to future positions.

7. Logits. After the final Transformer layer, a linear projection maps the last token's representation from d_model to vocab_size, producing raw unnormalized scores called logits — one score per vocabulary token.

logits = W_lm · h_last, W_lm ∈ ℝ^(vocab_size × d_model)

8. Softmax. Logits are converted to a probability distribution over the vocabulary via softmax.

9. Sampling. A token is selected from this distribution. The method matters enormously for output quality:

Temperature Scales logits before softmax. Low temp (→0) makes distribution sharper (greedy); high temp (→∞) flattens it (more random). Typical values: 0.7–1.0.
Top-k Restrict sampling to the k most probable tokens. Prevents low-probability noise tokens from being selected.
Top-p (Nucleus) Sample from the smallest set of tokens whose cumulative probability ≥ p. Adaptive: uses fewer tokens when distribution is sharp, more when flat.
Greedy Decoding Always pick the argmax token. Fast but prone to degenerate repetition. Not recommended for open-ended generation.

10. Autoregressive Generation. The sampled token is appended to the input sequence, and the entire process repeats until an end-of-sequence token is generated or a maximum length is reached. This is O(n²) in sequence length due to the attention matrix, which is a fundamental constraint of Transformer inference.

§ 06

Training Mechanics

Pretraining

Pretraining is the first and most expensive phase. The model is trained on a massive corpus (trillions of tokens from web crawls, books, code repositories, scientific papers) to minimize the next-token prediction loss. This is where the model acquires linguistic competence, world knowledge, and reasoning patterns.

Loss Function: Cross-Entropy

The training objective is to maximize the log-probability of the correct next token at every position. Equivalently, minimize cross-entropy loss between the predicted distribution and the one-hot target. For a sequence of T tokens: loss = −(1/T) × Σ log P(token_t | token_{1..t-1}).

Backpropagation and Gradient Descent

After computing the loss, backpropagation computes the gradient of the loss with respect to every parameter via the chain rule. An optimizer then updates all weights in the direction that decreases the loss. For LLMs with billions of parameters, the optimizer state itself (first and second moment estimates for Adam) can consume more memory than the weights.

Adam maintains per-parameter adaptive learning rates using exponential moving averages of gradients and their squares. AdamW decouples weight decay from the gradient update, which improves regularization and is the standard choice for LLM training.

Fine-Tuning and Alignment

Supervised Fine-Tuning (SFT) continues training on curated task-specific data. Instruction Tuning fine-tunes on instruction-response pairs to make the model follow natural-language directions. RLHF (Reinforcement Learning from Human Feedback) trains a reward model on human preference judgments, then optimizes the LLM's policy via PPO to maximize expected reward while minimizing KL divergence from the SFT baseline — preventing the model from "reward hacking" into incoherent outputs.

Scaling Laws

Chinchilla scaling laws (Hoffmann et al., 2022) show that model performance follows a power law with respect to compute, parameters, and training tokens, with an optimal trade-off: for a given compute budget, N parameters ≈ D/20 training tokens. Many early models were undertrained relative to their parameter count; Chinchilla demonstrated that smaller models trained on more data outperform larger models trained for fewer steps.

Distributed Training

Training GPT-scale models requires distributing computation across thousands of GPUs or TPUs. Data parallelism replicates the model across devices, each processing different mini-batches, then aggregates gradients. Tensor parallelism splits individual weight matrices across devices. Pipeline parallelism assigns different Transformer layers to different devices, passing activations between them. Modern frameworks like Megatron-LM combine all three.

§ 07

Modern LLM Concepts and Jargon

Prompting Strategies

Zero-shot prompting asks the model to perform a task with no examples — it relies entirely on knowledge from pretraining. Few-shot prompting provides examples within the context window, which serves as implicit in-context learning. Chain-of-thought prompting encourages the model to produce intermediate reasoning steps before the final answer, substantially improving performance on multi-step problems.

Context Window

The maximum number of tokens the model can process in a single forward pass. This is a hard architectural constraint. GPT-3's context was 4K tokens; modern models extend to 128K–1M tokens through techniques like RoPE scaling and FlashAttention. Long contexts increase the O(n²) attention cost significantly.

Hallucination and Grounding

LLMs generate fluent, confident-sounding text even when fabricating facts. This is a fundamental property of probabilistic next-token prediction — the model doesn't have a truth oracle. Grounding techniques constrain generation to verified sources. Retrieval-Augmented Generation (RAG) addresses this by retrieving relevant documents at inference time and injecting them into the prompt context, anchoring the model's output to external knowledge.

RAG and Vector Databases

A RAG pipeline: (1) encode the user query into an embedding vector; (2) perform approximate nearest-neighbor search in a vector database (e.g., Pinecone, Weaviate, pgvector) to retrieve semantically similar documents; (3) inject retrieved text into the prompt; (4) generate a response grounded in retrieved context. The quality of the embedding model and the retrieval architecture are as important as the generation model.

Agents and Tool Calling

LLMs can be augmented with tools: functions the model can invoke to take actions (API calls, code execution, web search, database queries). The model generates structured tool-call arguments, receives the tool's output, and continues reasoning. Multi-step agent systems chain multiple tool calls, planning and re-evaluating based on intermediate results — turning the LLM into a reasoning controller over external systems.

Architectural Advances

Mixture of Experts (MoE) replaces dense FFN layers with a collection of "expert" FFNs, routing each token to a small subset (e.g., 2 of 64 experts) via a learned gating network. This dramatically increases effective model capacity with sublinear compute cost. Mixtral-8x7B has 46B parameters but uses ~12B per forward pass. Multimodal models (GPT-4V, Gemini) encode non-text modalities (images, audio) into embedding spaces compatible with the text transformer.

Efficiency Techniques

Quantization reduces weight precision from FP32/BF16 to INT8 or INT4, shrinking model size and accelerating inference with minimal quality loss. Distillation trains a smaller student model to mimic a larger teacher model's output distribution. LoRA (Low-Rank Adaptation) freezes pretrained weights and injects trainable low-rank decomposition matrices into attention layers — dramatically reducing the number of trainable parameters for fine-tuning, making it feasible on commodity hardware. LoRA is the dominant PEFT (Parameter-Efficient Fine-Tuning) method.

§ 08

Inference Optimization

KV Cache

During autoregressive generation, the Key and Value projections for all previous tokens are identical across steps — recomputing them is wasteful. The KV cache stores K and V tensors for all past positions, requiring only a single new row to be computed per step. This reduces per-step compute from O(n²) to O(n) at the cost of memory: KV cache for long sequences can consume gigabytes.

Batching and Parallelism

Static batching groups fixed-size requests together. Continuous batching (dynamic batching) treats inference as a pipeline, inserting new requests into the batch as slots free up from completed sequences. This dramatically improves GPU utilization. Tensor parallelism at inference splits large weight matrices across multiple GPUs. For serving very large models, pipeline parallelism assigns layer groups to different devices.

Latency vs. Throughput

These are in fundamental tension. Increasing batch size improves throughput (tokens/second/GPU) but increases time-to-first-token for individual requests. Speculative decoding (a small draft model generates token candidates; the large model verifies in parallel) can reduce latency without sacrificing quality. Token streaming — yielding tokens to the client as they're generated — reduces perceived latency by avoiding waiting for full completion.

End-to-End System Architecture

A production LLM deployment is not just a model — it's a layered system spanning API infrastructure, retrieval, safety, routing, and caching components working in concert.

Each component is independently scalable and replaceable. The retrieval layer can be swapped without touching the model. The safety filters can be updated with policy changes. The model can be swapped via canary deployment. This layered design is what separates "running a model" from "deploying an LLM system."

§ 10

How an LLM Generates a Response — End-to-End Walkthrough

Let's trace exactly what happens from the moment you press Enter to the moment you see the first word of a response. The prompt is: "Explain the Pythagorean theorem in simple terms."

01
API receives the request

The raw string is sent via HTTP/WebSocket to the inference endpoint. Authentication, rate limiting, and input validation are applied. A request ID is assigned.

02
Safety pre-filter

A fast classifier checks the input for disallowed content — jailbreak patterns, sensitive PII, policy violations. If flagged, the request is rejected before ever reaching the model.

03
Context assembly

The system prompt (if any), retrieved documents (if RAG is active), conversation history, and the user's message are concatenated into the full context window. Token budget is calculated.

04
Tokenization

The assembled string is fed to the tokenizer. "Pythagorean" becomes multiple tokens. The output is a tensor of integer IDs of shape (1 × seq_len).

05
Embedding lookup + positional encoding

Token IDs are passed through the embedding matrix → (seq_len × d_model). Positional encodings are added. The sequence now lives in continuous high-dimensional space.

06
N Transformer layers execute

Each layer runs multi-head self-attention over the full sequence, computes attention scores, weighs Values, adds residuals, normalizes, passes through FFN. Contextual representations deepen with each layer.

07
Logit projection + sampling

The final hidden state at the last position is projected to vocab_size logits. Temperature is applied. Top-p nucleus sampling selects the next token — e.g., token ID 464 = "The".

08
Token streamed to client

"The" is immediately sent to the client via server-sent events. The user sees the first word appear. Perceived latency drops to time-to-first-token.

09
Autoregressive loop

Token 464 is appended to the input. KV cache serves previous K/V tensors. A new forward pass computes the next token. This repeats — "The", "Pythagorean", "theorem", ... — until <EOS> or max_tokens.

10
Safety post-filter + delivery

The complete response passes through content moderation. If it clears, the connection is closed. Telemetry is logged: latency, token count, prompt fingerprint, output hash.

Key Takeaway

Every word you see is the product of a probability distribution sampled over 50,000+ candidates, shaped by billions of parameters trained on trillions of tokens, executed in milliseconds on specialized silicon. The model doesn't "know" the Pythagorean theorem — it has learned that given this context, these tokens have high probability of following one another in a coherent and accurate way.
