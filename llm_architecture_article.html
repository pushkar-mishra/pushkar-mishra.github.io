1. Foundations of AI and LLMs

The Hierarchy: AI → ML → Deep Learning → Generative AI

These terms are often used interchangeably but represent a nested hierarchy. Artificial Intelligence (AI) is the broadest umbrella — any technique enabling machines to exhibit intelligent behavior, including rule-based systems, search, and expert systems. Machine Learning (ML) is a subset where systems learn patterns from data rather than explicit rules. Deep Learning (DL) uses layered neural networks to learn hierarchical representations. Generative AI produces new content — text, code, images — by learning the underlying distribution of training data.

🤖 Artificial Intelligence

Any technique that enables machine intelligent behavior. Includes rule-based systems, expert systems, planning engines.

📊 Machine Learning

Systems that learn patterns from data without explicit programming. Requires labeled or structured training signal.

🧬 Deep Learning

Multi-layered neural networks that learn hierarchical feature representations automatically from raw data.

✨ Generative AI

Models that learn a data distribution and can sample new, coherent content — text, images, code, audio.

Neural Networks: The Foundation

A neural network is a computation graph of parameterized linear transformations followed by nonlinear activations. Each neuron computes a weighted sum of its inputs plus a bias, then applies an activation like ReLU or GELU. Stacking layers allows the network to learn progressively abstract features: pixels → edges → shapes → objects; or characters → words → phrases → meaning.

output = activation(W · input + b) W ∈ ℝ^(output_dim × input_dim), b ∈ ℝ^output_dim ReLU(x) = max(0, x) GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³)))

Training vs. Inference

Training is the optimization phase — given data and a loss function, model weights are iteratively updated via gradient descent to minimize prediction error. It's compute-intensive and done offline. Inference is the deployment phase — with fixed weights, the model predicts on new inputs in real time. The distinction matters because they have very different hardware requirements, latency constraints, and optimization targets.

Learning Paradigms

Supervised learning trains on labeled input-output pairs. Unsupervised learning finds structure in unlabeled data. Self-supervised learning — the paradigm powering LLMs — generates its own supervision signal from raw data. For language models, this means predicting the next token from all previous tokens, using the text itself as the label, enabling training on internet-scale corpora without manual annotation.

2. Transformer Architecture

Why Transformers? The Failure of RNNs

Before Transformers, sequence modeling relied on RNNs, LSTMs, and GRUs — models that process tokens sequentially and maintain a hidden state. The problems were fundamental: vanishing/exploding gradients made learning long-range dependencies difficult; sequential processing prevented GPU parallelization; and the fixed hidden state bottlenecked information flow over long sequences.

The 2017 paper "Attention Is All You Need" (Vaswani et al., Google Brain) proposed replacing recurrence entirely with attention mechanisms — letting every token attend directly to every other token, enabling full parallelization and constant-length dependency paths regardless of sequence length.

The Meeting Notes Analogy

An RNN reads a document like someone taking meeting notes sequentially — by the time they reach page 10, early details from page 1 are hazy. A Transformer is like having the entire meeting transcript on a table and being able to instantly highlight any connection between any two moments — no sequential bottleneck, no fading memory.

Encoder-Decoder Architecture

The original Transformer was designed for sequence-to-sequence tasks like translation. The Encoder processes the full input and produces rich contextual representations. The Decoder autoregressively generates the output, attending to both its own previous outputs (self-attention) and the encoder's representations (cross-attention). Modern generative LLMs strip out the encoder and use decoder-only architectures.

Self-Attention: The Core Mechanism

For every token, three vectors are computed from its embedding via learned linear projections: a Query (Q), a Key (K), and a Value (V). Think of it as a soft database lookup — the Query is what you're looking for, Keys are index entries, and Values are the content retrieved.

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V Q = X · Wq, K = X · Wk, V = X · Wv Wq, Wk, Wv ∈ ℝ^(d_model × d_k)

Dividing by √d_k prevents dot products from growing too large in high-dimensional space (which would saturate the softmax and collapse gradients). The softmax converts raw scores to a probability distribution; the weighted sum over V produces a context-aware representation per token.

Multi-Head Attention: Learning Multiple Relationship Types

Running H parallel attention heads — each with independent Q/K/V projections — lets the model simultaneously capture different relationship types: one head may track subject-verb agreement, another resolves pronoun coreference, another detects positional proximity. Outputs are concatenated and projected: MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) · Wᴼ

Positional Encoding

Self-attention is permutation-invariant — it sees the input as a set, not a sequence. Positional encodings are added to token embeddings before the first layer to inject order information. The original paper used sinusoidal functions of varying frequencies. Modern models use learned encodings or Rotary Position Embeddings (RoPE), which encode relative positions directly into attention computations and generalize better to longer sequences than seen during training.

Feed-Forward Networks & Residual Connections

Each Transformer block contains a two-layer FFN applied identically at every token position — this is where much of the model's "factual knowledge" is stored. Research suggests FFN layers function as key-value memories. The FFN hidden dimension is typically 4× the model dimension, making it the dominant source of parameters. Both the attention and FFN sublayers are wrapped in residual (skip) connections to preserve gradient flow, followed by Layer Normalization to stabilize activations.

3. Variants of Transformer Models

Not all Transformers are the same. Three architectural families emerged with distinct training objectives and ideal use cases:

🔎 Encoder-only (e.g., BERT)

Bidirectional attention — every token sees all others. Trained with Masked Language Modeling (predict randomly masked tokens). Best for classification, NER, semantic search, embeddings.

📝 Decoder-only (e.g., GPT, LLaMA, Mistral)

Causal (left-to-right) attention — each token sees only previous tokens. Trained with Next Token Prediction. The dominant architecture for generative LLMs.

🔄 Encoder-Decoder (e.g., T5, BART)

Full encoder reads input; decoder generates output attending via cross-attention to encoder states. Optimal for translation, summarization, question answering.

⚡ Autoregressive vs Masked

Autoregressive generates left-to-right conditioning on previous tokens. Masked models use bidirectional context. Autoregressive is standard for text generation tasks.

4. Tokenization and Vector Representation

Tokens vs. Words

LLMs don't process words or characters — they operate on tokens, subword units that balance vocabulary coverage against size. "unbelievable" might tokenize as ["un", "believ", "able"]; "GPT-4" as ["G", "PT", "-", "4"]. Typical vocabularies contain 32,000–128,000 tokens. The tokenizer is a critical, often underappreciated component — it profoundly affects performance on code, numbers, and non-Latin scripts.

🔤 Byte Pair Encoding (BPE)

Starts with characters, iteratively merges the most frequent adjacent pairs. Used by GPT-2, GPT-3, GPT-4, Llama 3. Efficient and language-agnostic.

🧩 WordPiece

Similar to BPE but merges that maximize training data likelihood under a language model. Used by BERT, DistilBERT.

🌐 SentencePiece

Works directly on raw text without pre-tokenization, making it robust to whitespace variations. Used by T5, LLaMA 1 & 2, Gemma.

Embeddings: Tokens in Vector Space

Each token ID is mapped to a dense vector via an embedding lookup table — a matrix of shape (vocab_size × d_model). This converts discrete IDs to continuous vectors the model can reason about mathematically. For GPT-3, d_model = 12,288. For smaller models like GPT-2 small, d_model = 768.

The GPS Coordinate Analogy

Think of the embedding matrix as a dictionary where every word has a GPS coordinate in high-dimensional space. Words with similar meanings cluster nearby; semantic relationships become vector arithmetic — "king − man + woman ≈ queen". The model navigates this space to reason about meaning.

Word Embeddings vs. Contextual Embeddings

Classical embeddings like Word2Vec assign one fixed vector per word regardless of context — "bank" gets the same vector whether it means a riverbank or a financial institution. In Transformers, embeddings are contextual: the representation of a token is dynamically shaped by every other token in the sequence via attention. The same word at different positions in different sentences produces fundamentally different internal representations.

5. Data Flow Inside an LLM

Here is exactly what happens from raw text input to generated output, step by step:

The Forward Pass: Step-by-Step

1. Input Prompt — Raw text string from the user.
2. Tokenization — String → sequence of integer token IDs. "Hello world" → [9906, 995].
3. Embedding Lookup — Each ID fetched from embedding matrix → shape (seq_len × d_model).
4. Positional Encoding — Position vectors added element-wise, injecting sequence order.
5. N Transformer Layers — Multi-head attention + FFN, repeated N times (e.g., 96× for GPT-3).
6. Attention Computation — Q/K/V projected, attention matrix computed, causal mask applied, Values weighted.
7. Logit Projection — Final hidden state projected to vocab_size raw scores.
8. Softmax — Logits → probability distribution over vocabulary.
9. Sampling — A token is selected; appended to sequence.
10. Repeat — Autoregressive loop continues until EOS or max_tokens.

Sampling Strategies

🌡️ Temperature

Scales logits before softmax. Low temp (→0) → greedy/sharp distribution. High temp (→∞) → flat/random. Typical: 0.7–1.0.

🎯 Top-k Sampling

Restrict sampling to the k most probable tokens. Prevents rare, low-probability tokens from being selected.

🔵 Top-p (Nucleus)

Sample from the smallest set of tokens summing to ≥ p cumulative probability. Adaptive: narrow when confident, broad when uncertain.

🏆 Greedy Decoding

Always pick argmax token. Deterministic and fast but prone to repetitive, degenerate output. Not recommended for open-ended generation.

logits = W_lm · h_last, W_lm ∈ ℝ^(vocab_size × d_model) P(token) = softmax(logits / temperature) # Top-p nucleus sampling: sorted_probs = sort(P, descending) cumulative_probs = cumsum(sorted_probs) nucleus = tokens where cumulative_probs ≤ p next_token = sample(nucleus)

6. Training Mechanics

Pretraining

The first and most expensive phase. The model trains on a massive corpus — trillions of tokens from web crawls, books, code, papers — to minimize next-token prediction loss. This is where the model acquires linguistic competence, factual knowledge, and reasoning patterns. GPT-3 was trained on ~300B tokens; modern models use 1–15 trillion tokens.

Loss Function: Cross-Entropy

The objective is to maximize the log-probability of the correct next token at every position. Equivalently, minimize cross-entropy loss between the predicted probability distribution and the one-hot ground-truth label:

Loss = -(1/T) × Σ log P(token_t | token_{1..t-1}) # For each position t in sequence of length T: # The model predicts P over all vocab tokens # Loss penalizes low probability assigned to the correct token

Backpropagation & Optimizers

After computing loss, backpropagation applies the chain rule to compute gradients of the loss with respect to every parameter. An optimizer then updates weights. Adam maintains per-parameter adaptive learning rates using exponential moving averages of first (mean) and second (variance) gradient moments. AdamW decouples weight decay from the gradient update — the standard choice for LLM pretraining. The optimizer state itself (moment estimates for all parameters) often consumes more memory than the weights.

Fine-tuning and Alignment Pipeline

📚 Supervised Fine-Tuning (SFT)

Continue training on curated, high-quality task-specific examples. Adapts the base model to follow formats and domain conventions.

📋 Instruction Tuning

Fine-tune on instruction-response pairs so the model reliably follows natural-language directions rather than just completing text.

🏅 RLHF

Train a reward model on human preference judgments. Optimize the LLM via PPO to maximize expected reward while constraining deviation from the SFT baseline (KL penalty).

📈 Scaling Laws

Chinchilla (2022): for a compute budget C, optimal N parameters ≈ D/20 tokens. Many early models were overtrained on parameters, undertrained on data.

The Apprenticeship Analogy for RLHF

Pretraining is like reading every book ever written. SFT is an apprenticeship with a master craftsman. RLHF is iterative feedback from real customers — the model learns not just what is technically correct but what humans actually prefer, rate highly, and find useful.

Distributed Training

Training GPT-scale models requires distributing across thousands of GPUs or TPUs. Data parallelism replicates the model across devices, each processing different batches, then aggregates gradients. Tensor parallelism splits weight matrices across devices. Pipeline parallelism assigns different Transformer layers to different devices. Frameworks like Megatron-LM combine all three strategies (3D parallelism).

7. Modern LLM Concepts and Jargon

Prompting Strategies

Zero-shot prompting asks the model to perform a task with no examples, relying entirely on pretrained knowledge. Few-shot provides examples in the context window as implicit in-context learning. Chain-of-thought prompting encourages intermediate reasoning steps before the final answer, dramatically improving multi-step problem performance — especially for math and logic.

Context Window

The maximum token sequence a model can process in one forward pass — a hard architectural constraint. GPT-3 had a 4K context; modern models extend to 128K–1M tokens via RoPE scaling, FlashAttention, and sparse attention patterns. Long contexts increase the O(n²) attention computation cost significantly.

Hallucination and Grounding

LLMs generate fluent, confident output even when fabricating facts — a fundamental property of probabilistic next-token prediction. The model has no truth oracle. Grounding constrains generation to verified external sources. Retrieval-Augmented Generation (RAG) retrieves relevant documents at inference time and injects them into context, anchoring output to external knowledge.

How RAG Works in Practice

1. Encode the user query into an embedding vector using a retrieval model.
2. Run approximate nearest-neighbor (ANN) search against a vector database (Pinecone, Weaviate, pgvector) to find semantically similar documents.
3. Inject retrieved chunks into the prompt as additional context.
4. The LLM generates a response grounded in retrieved facts, with citations.
The quality of the embedding model and retrieval architecture are as critical as the generation model itself.

Advanced Architecture Techniques

🤝 Agents & Tool Calling

LLMs invoke external tools — APIs, code execution, web search, databases. The model generates structured arguments, receives results, and continues reasoning in a loop.

🎛️ Mixture of Experts (MoE)

Replaces dense FFN with many "expert" sub-networks. A router selects 2–8 experts per token. More capacity at sub-linear compute cost. Used in Mixtral, GPT-4 (reportedly).

🗜️ Quantization

Reduce weight precision from FP32/BF16 to INT8 or INT4. Shrinks model size and accelerates inference with minimal quality degradation. GPTQ, AWQ are popular methods.

🎓 Distillation

Train a small "student" model to mimic a large "teacher" model's output distribution — not just hard labels. Produces efficient models that punch above their size.

🔧 LoRA (Low-Rank Adaptation)

Freeze pretrained weights; inject trainable low-rank decomposition matrices into attention layers. Reduces trainable parameters by 100–10,000×. Dominant PEFT method.

🖼️ Multimodal Models

Encode images, audio, video into embedding spaces compatible with the text Transformer. The same attention mechanism operates over mixed-modality token sequences.

The LoRA Analogy

Fine-tuning a 70B model is like renovating an entire skyscraper. LoRA is like adding a modular extension wing — you keep the main building intact, build a small efficient add-on that adjusts the overall output, and can swap different wings for different tenants (tasks). The skyscraper doesn't change; only the extension does.

8. Inference Optimization

KV Cache

During autoregressive generation, the Key and Value projections for all previous tokens are identical at each step — recomputing them wastes cycles. The KV cache stores K and V tensors for all past positions; only a single new row is computed per step. This reduces per-step compute from O(n) to O(1) additional work, at the cost of memory. For long sequences (128K tokens), KV cache can consume tens of gigabytes.

Batching Strategies

Static batching groups requests of identical length — simple but wastes compute on padding. Continuous (dynamic) batching treats inference as a pipeline, inserting new requests as slots free up from completed sequences. This is the primary technique for maximizing GPU utilization in production LLM serving, and is implemented in frameworks like vLLM and TensorRT-LLM.

⚡ Tensor Parallelism

Split large weight matrices across multiple GPUs. Each GPU holds a shard; results are all-reduced. Reduces per-device memory and latency for large models.

📦 Pipeline Parallelism

Assign different layer groups to different GPUs. Micro-batching reduces pipeline bubble overhead. Enables models too large to fit on a single node.

🏎️ Speculative Decoding

A small draft model generates token candidates in parallel; the large model verifies them in one pass. Achieves 2–3× latency reduction without quality loss.

🌊 Token Streaming

Yield tokens to the client as they're generated (server-sent events). Reduces perceived latency — users see the first word immediately rather than waiting for full completion.

Each component is independently deployable and replaceable. The retrieval layer can be swapped without touching the model. Safety filters update with policy changes. The model can be rolled out via canary deployment. This layered design separates "running a model" from "operating an LLM product."

10. How an LLM Generates a Response — End-to-End Walkthrough

Let's trace exactly what happens when you send the prompt: "Explain the Pythagorean theorem in simple terms."

  1. API receives the request

    The raw string arrives via HTTP POST. Authentication, rate limiting, and input validation are applied. A request ID and timestamp are assigned. The request is queued for preprocessing.

  2. Safety pre-filter

    A fast classifier checks for disallowed content — jailbreak patterns, sensitive PII, policy violations. If flagged, the request is rejected before reaching the model. This runs in ~5ms and saves expensive GPU cycles.

  3. Context assembly

    System prompt (if any), retrieved documents (if RAG is active), conversation history, and the user message are concatenated. Token budget is calculated to stay within the context window limit.

  4. Tokenization

    The assembled string is passed to the tokenizer. "Pythagorean" becomes multiple tokens. Output: a tensor of integer IDs, e.g., [1, 849, 2480, 262, 47363, 25266, 1013, 10688, 2063, 13].

  5. Embedding lookup + positional encoding

    Each token ID is fetched from the embedding matrix → shape (seq_len × 12288). Positional encodings are added element-wise. The sequence now lives in continuous 12,288-dimensional space.

  6. 96 Transformer layers execute (for GPT-3 scale)

    Each layer runs multi-head attention over the full sequence, computes attention scores, applies causal mask (blocking future positions), weights Values, adds residuals, normalizes, passes through FFN. Representations deepen and contextualize with each layer.

  7. Logit projection + temperature + sampling

    The final hidden state at the last position is projected to vocab_size logits. Temperature 0.8 is applied. Top-p=0.95 nucleus sampling selects the next token — e.g., token 464 = "The".

  8. Token streamed to client

    "The" is immediately sent via server-sent events. The user sees the first word appear on screen. Time-to-first-token is the critical latency metric — typically 200–800ms.

  9. Autoregressive loop

    Token 464 is appended to the input sequence. The KV cache serves all previous K/V tensors — only one new row is computed. Next forward pass produces "Pythagorean", then "theorem", and so on until <EOS> or max_tokens is reached.

  10. Safety post-filter + delivery complete

    The full response passes content moderation. If it clears, the stream is closed. Telemetry is logged: latency, token count, prompt fingerprint, cost, output hash. The cycle is complete.

The Final Perspective

Every word you see is the product of a probability distribution sampled over 50,000+ candidates, shaped by hundreds of billions of parameters trained on trillions of tokens, executed in milliseconds on specialized silicon. The model doesn't "know" the Pythagorean theorem the way you do — it has learned that given this context, these specific tokens have high probability of following one another in a way that is coherent, accurate, and useful. The intelligence is statistical, emergent, and deeply distributed across every weight in the network.

"Language models are few-shot learners — and the more you understand their architecture, the better you can engineer both the models and the prompts that guide them."
